# [프로젝트 명세서 Aico Buddy

## 1. 프로젝트 개요
* **프로젝트명**: 낭비량 시각화와 남용 방지 캠페인을 활용한 친환경 AI 이용 서비스, 'AIco (AI+Eco) Buddy'
* **핵심 요약**: 생성형 AI 사용 시 발생하는 탄소 배출과 수자원 낭비를 줄이기 위한 저탄소 AI 채팅 웹앱
* **주요 타겟**: 대학생 및 학내 AI 이용자
* **핵심 가치**: "AI 가성비 있게 쓰는 방법" (비용 절감 및 환경 보호)

---

## 2. 시스템 아키텍처 및 백엔드 로직 (AI Agent 개발 핵심)

### 2.1 난이도별 라우팅 백엔드 조건문 (Routing Logic)
* **소형 모델(SLM) 분산**: 사용자가 입력한 질문의 복잡도와 난이도가 낮을 경우 경량 모델로 자동 분류하여 제안하기 (Gemini, ChatGPT, Claude Web에서 이용 가능하도록)
* **거대 모델(LLM) 분산**: 고난도 추론이 필요한 복잡한 질문만 무거운 모델로 처리
* **효과**: 불필요하게 복잡한 추론 모델 작동을 방지하여 토큰 및 전력 소모 최소화

### 2.2 프롬프트 최적화 및 필터링 엔진 기능
* **질문 압축**: 입력된 프롬프트를 분석하여 최소한의 토큰으로 처리할 수 있도록 최적화(압축)
* **단순 요청 기각**: 지나치게 단순하거나 무의미한 요청의 경우, AI가 직접 처리하는 대신 기각하거나 대체 방법 안내
* **중복 질문 방지**: 사용자의 이전 질문 데이터 내역을 정리 및 캐싱하여 중복 질문 유입 차단

### 2.3 데이터 처리 (on-device 이용)
* **라이브러리**: 토큰 계산 라이브러리(Tiktoken 등) 활용을 통한 실시간 토큰 소모량 검증 - 외부 통신하지 말고 on-device 계산 매커니즘으로 수행
* **산출 공식 정밀화**: 학술 자료(IEA 등)를 기반으로 AI 쿼리 1회당 토큰-전력량-냉각수 낭비량의 상관관계 및 정량적 기준 공식 수립

## 3. 프론트엔드 UI/UX (대시보드 기능) - 가장 중요 !! 이것을 우선으로 개발.

### 3.1 실시간 양방향 대시보드 웹 화면
* **절감 지표 (Green)**: 사용자 행동(질문 압축, SLM 라우팅 등)으로 인해 절약된 에너지, 토큰, 디지털 탄소 배출량 시각화
* **낭비 지표 (Red)**: 절감 노력에도 불구하고 생성형 AI를 사용하면서 여전히 소비/낭비된 기회비용적 탄소량 및 냉각수 병렬 제시
* **시각화 형태**: 보고서 형태의 레이아웃을 통해 사용자가 환경에 미치는 영향력을 직관적이고 균형 있게 인식하도록 유도

## 4. 최종 산출물 및 마일스톤

### 4.1 최종 개발 결과물
1.  **웹 브라우저 Extension**: 양방향 대시보드 기반 저탄소 AI 채팅 웹앱
2.  **리포트 시스템**: 베타 테스트 결과 기반, 캠퍼스 내 실제 감축한 탄소 배출량 및 데이터 용량 정량적 성과 증명 리포트

### 4.2 피어멘토링 반영 보완 과제 (우선순위 반영)
* **아이러니 해결**: 토큰을 절약하기 위해 최적화 로직(Agent)이 추가 토큰을 과도하게 소비하는 문제 방지 설계 필요
* **기능 집중**: 시각화는 부수적 기능으로 두고, 위의 **라우팅 로직 및 프롬프트 최적화 엔진 개발**에 개발 시간 집중
